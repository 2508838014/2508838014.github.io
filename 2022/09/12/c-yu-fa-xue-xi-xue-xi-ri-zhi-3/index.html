8.一些杂谈 耦合与内聚
软件要做到高内聚，低耦合。
内聚就是一个模块内部各成分之间相关联的成都的度量
耦合就是程序结构中哥哥模块的相互关联的度量

内聚的分类：由高到低

内聚的分类：由高到低
功能内聚：知模块内的所有元素共同作用完成一个功能，缺一不可。
顺序内聚：一个模块中的哥哥处理元素都密切像关于同一个功能且必须顺序执行，迁移功能元素的输出就是下一个功能元素的输入。
通信内聚：模块内所有处理元素都在同一个数据结构中。
过程内聚：一个模块完成多个任务，这些任务必须按照指定的过程执行。
瞬时内聚：把需要同时执行的任务或者动作组合在一起（例如初始化模块）
逻辑内聚：模块完成逻辑上相关的一组任务。
偶然内聚：一个模块内的各个处理元素之间没有任何联系或者有松散的联系

耦合的分类：由高到低

耦合的分类：由高到低
内容耦合：一个模块直接使用另一个模块的内部数据，或者通过非正常入口转入另个一模块内部是，这种耦合关系叫做内容耦合
公共耦合:一组模块访问一个公共的数据环境，如全局数据变量
外部耦合：一组模块访问一个公共变量，这里指的是基本数据类型而不是数据结构躲着说是对象。
控制耦合：一个模块调用另一个模块的时候，传递的是控制变量，被调用模块通过该控制变量的值选择执行模块内莫伊功能，那么也就是说被调用的模块应该具有多个功能。
标记耦合：耦合模块之间以数据结构传递（比如在java中，传递就是一个对象）
数据耦合：耦合模块之间有调用关系，传递的是简单数据类型的值。
无直接耦合：两个模块之间没有直接的关系，他们从属于不同模块的控制与调用，它们之间不传递任何信息。

一、委托

委托两类：C#类提供的，自定义委托

C#提供的委托（无参数与返回值）action

自定义个类与方法

class Calculator
{
	public void Report()
	{
		Console.WriteLinr("I have 3 methods");
	}
	public int ADD(int a,int b)
	{
		int result = a+b;
		return result;
	}
	public int SUB(int a,int b)
	{
		int result = a-b;
		return result;
	}
}

调用方法
三种调用结果相同
第一种没有使用委托
后两种可以是基本相同

static void Main(string {} args)
{
	Calulator calculator new Calculator();
	//acton只想方法
	Action action = new Action(calculator.Report);
	//以下三种方法调用结果相同
	//直接调用
	calculator.Report();
	//使用委托间接调用
	action.Invoke();
	//Invoke也可以省略掉
	action();
}

C#提供的委托（有参数与返回值）function

调用方法

static void Main(string {} args)
{
	Calulator calculator new Calculator();
	Func<int,int,int> fun1 = new Fun<int,int,int>(calculator.ADD);

C#自定义委托

委托就是一种类
注意声明委托的位置（一般写在命名空间下），可能会引起嵌套类
委托与所封装的方法必须“类型兼容”
返回值与参数列表的数据类型需要一致
以下是委托的声明

public delegate int Calc(int x, int y);

以下是实现委托的类 

clas Calculator
{
	public int Add(int a, int b)
	{
		retutn a+b;
	}
	public int Sub(int a,int b)
	{
		return a-b;
	}
}

以下是委托的具体实现
具不具有Invoke都是一样的。 

class Program
{
	static void Main(string[] args)
	{
		Calculator calculator = new Calculator():
		Calc calc1= new Calc(calculator.Add(int a,int b));
		Calc calc2= new Calc(calculator.Sub(int a,int b));


		int a=10;
		int b=30;
		int c=0;


		c=calc1.Invoke(a,b);
		Console.WriteLine(c);
		c=calc2.Invoke(a,b);
		Console.WriteLine(c);

		c=calc2(a,b);
		Console.WriteLine(c);

	}
}

委托的一般使用

一般使用方法把方法当作参数传给另一个方法
这又可以分为两种用法
方法1 模板方法
借用指定的外部方法来产生结果

  1. 相当于“填空题”
  2. 常位于代码中部
  3. 常用于委托有返回值
    方法2 回调方法（callback），调用指定的外部方法
  4. 相当于“流水线”
  5. 常位于代码末尾
  6. 常委托无返回值

委托的缺陷

  1. 方法级别的紧耦合，工作中要慎之又慎
  2. 使可读性下降，debug的难度增加
  3. 把委托回调，异步调用和多线程纠缠在一起，会让代码变得难以维护与阅读
  4. 委托使用不当有可能造成内存泄漏和程序性能下降
